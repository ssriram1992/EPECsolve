8 # Prerequisites
9 
10 ## Mandatory packages
11 - [Armadillo](http://arma.sourceforge.net/) (Version 9.6 or later recommended. Minimum 8.5 required.)
12  * BLAS
13  * ARPACK
14  * LAPACK
15 - [Gurobi](https://www.gurobi.com/registration/download-reg) (Version 8.1 or later)
16 - [gcc/g++](https://gcc.gnu.org/) (Tested on version 4.8. Must support C++11 and be compatible with your version of Gurobi) `sudo apt install gcc ` will install gcc/g++ on an Ubuntu machine.
17 - [GNU make](https://www.gnu.org/software/make/) `sudo apt install make` will install GNU make on an Ubuntu machine.
18 - [Boost](https://www.boost.org/) Required for logging, commandline interface to solve files etc. Can produce a boost-free version if there is significant interest.
19 
20 ## Recommended but not mandatory for the algorithm. (Some examples might have these dependancies)
21 - [Rapid JSON](http://rapidjson.org/) To export results and save example problem instances.
22 - [DOxygen](http://www.doxygen.nl) Only if you need documentation. `sudo apt install doxygen ` will install DOxygen on an Ubuntu machine.

24 # Getting the documentation
25 One can generate two versions of documentation for this project..
26 - Use the simple version of documentation, if you are only interested in using this as a predefined library which you don't intend to edit. 36 # Compiling
37 - Download the [project](https://github.com/ssriram1992/EPECsolve/). If you do not have access, please email [sriram.sankaranarayanan@polymtl.ca](mailto:sriram.sankaranarayanan@polymtl.ca).

39 ## Makefile
40 - Open `Makefile`.
41 - Enter the path where you downladed in `EPEC_HOME`. This folder should contain folders like `docs/`, `src/`, `test/` etc.
42 - Enter the path to Boost in `BOOST_HOME`. Ensure that the path to corresponding include files and boost libraries are correct.
43 - Enter the path of your armadillo-installation in the line defining the variable `ARMA`. Typically, the location would be like `/opt/armadillo-code`.
44 - Enter the path of your Gurobi-installation in the line defining the variable `GUR`. Typically, the location would be like `/opt/gurobi/gurobi811/<Your OS>`.
45 - Run `make install` to make the necessary folders to store temporary files, binary files etc.
46 - Run `make compileEPEC` to compile.
47 - Optionally run `make EPECtest` to run the set of unit tests. It should all succeed without a problem.
48 - Run `make` to create the binary.
49 - Run `./bin/EPEC -h` to get a list of command line options with which you can run the executable.
50 
51 ## CMake
52 - Open `CMakeList.txt`.
53 - You will find two configurations. `ROSSOBIANCO` represents the remote server one, while the other is the local one.
54 - Set `BOOST_ROOT`, `ARMA_ROOT`, and `ARMA_LIB` to your customized folders.
55 - Set `GUROBI_PATH_%OS` to your local gurobi path (and version). Note that %OS corresponds to your local (or remote) operatin system.
