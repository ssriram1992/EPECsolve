3 # EPECsolve
4 Code to compute mixed-equilibrium in linear EPECs.
5 - [Code](https://github.com/ssriram1992/EPECsolve/)
6 - [Instances](https://github.com/ds4dm/EPECInstances) with detailed [mathematical description](https://github.com/ds4dm/EPECInstances/blob/master/Description.pdf)

40 ## Makefile
41 - Open `Makefile`.
42 - Enter the path where you downladed in `EPEC_HOME`. This folder should contain folders like `docs/`, `src/`, `test/` etc.
43 - Enter the path to Boost in `BOOST_HOME`. Ensure that the path to corresponding include files and boost libraries are correct.
44 - Enter the path of your armadillo-installation in the line defining the variable `ARMA`. Typically, the location would be like `/opt/armadillo-code`.
45 - Enter the path of your Gurobi-installation in the line defining the variable `GUR`. Typically, the location would be like `/opt/gurobi/gurobi811/<Your OS>`.
46 - Run `make install` to make the necessary folders to store temporary files, binary files etc.
47 - Run `make compileEPEC` to compile.
48 - Optionally run `make EPECtest` to run the set of unit tests. It should all succeed without a problem.
49 - Run `make` to create the binary.
50 - Run `./bin/EPEC -h` to get a list of command line options with which you can run the executable.
51 
52 ## CMake
53 - Open `CMakeList.txt`.
54 - You will find two configurations. `ROSSOBIANCO` represents the remote server one, while the other is the local one.
55 - Set `BOOST_ROOT`, `ARMA_ROOT`, and `ARMA_LIB` to your customized folders.
56 - Set `GUROBI_PATH_%OS` to your local gurobi path (and version). Note that %OS corresponds to your local (or remote) operatin system.

58 # Command-line interface
59 EPECSolve has a command line interface built on top of the standard modeling paradigm (see documentation for src/Models.cpp).
60 Once the executable is compiled via `make compileEPEC`, the user can run EPECSolve with: `./EPEC $options`.
61 
62 The following I/O options are available:
63 
64 * `-i (--input) PathToInstance`: This **required** option specifies the path to the JSON instance file. Extension .json is automatically appended to the argument.
65 * `-s (--solution) PathAndFile`: Sets the output (path) filename of the JSON solution file, if any (see option `-w. (writelevel)`. If a JSON output is required, extension .json is automatically appended to the argument.
66 * `-l (--log) PathAndFile`: Sets the output path/filename of the log file.
67 * `-w (--writelevel) intValue`: Sets the verbosity parameter for the output solution file. *0*: output only a JSON solution file (if any). *1* Only a human readable file. *2* Both.
68 * `-m (--message) intValue`: Sets the verbosity parameter for the program itself. *1*: only info messages, *2* info and debug, *3* info, debug and trace messages.
69 
70 
71 Algorithmic parameters:
72 
73 * `-a (--algorithm) intValue`: Specifies the type of algorithm employed to solve the instance. *0* is **fullEnumeration**, *1* is **innerApproximation**, *2* is **combinatorialPNE**. Note that the latter only works with pure equilibria.
74 * `-p (--pure) intValue`: Specifies whether the algorithm should seek for a pure equilibrium or not (note that **combinatorialPNE** will always seek for a pure solution) . *0* no requirement (either pure or mixed), *1* only pure.
75 * `-bigM intValue`: Specifies the whether LCP problems should be formulated with a LCP. *0* (default) for indicator constraints, *1* for bigM formulation.
76 * `-b (--bound) intValue`: Specifies if the final LCP model should be bounded by a *bigM* constant (see option `-boundBigM`) in their primal variables. *0* no bounding, *1* bounding with *bigM*.
77 * `-boundBigM intValue`: Specifies the *bigM* constant for the above param (see option `-b (--bound)`.
78 * `-t (--threads) intValue`: Specifies the number of threads Gurobi uses.
79 * `-tl (--timelimit) intValue`: Specifies the timeLimit (in seconds) for running the whole algorithmic procedure.
80 
81 Algorithmic parameters tailored on **innerApproximation**:
82 
83 * `-ad (--add) intValue`: Specifies the *EPECAddPolyMethod* strategy employed to expand the inner approximation when no information about deviations is available. *0* add polyhedra in lexographic order, *1* reverse lexographic, *2* random.
84 * `-ag (--aggr) intValue`: Specifies the *EPECAddPolyMethod* aggressivity (see option `-ad (--add)`). *int* specifies the number of lower-level polyhedra to be added to each Stackelberg leader at each iteration in which *EPECAddPolyMethod* is triggered.
85 * `-r (--recover) intValue`: Specifies the recover stragegy for the **innerApproximation** method. When the algorithm finds a mixed equilibrium and the parameter `-p (--pure)` is set to *1*, the algorithm will search for a pure equilibrium either by randomly expanding the inner approximation (*EPECAddPolyMethod*) or by triggering a **combinatorialPNE** procedure warmstarted with the current information. *0* sets the recover strategy to *incrementalEnumeration*, *1* sets it to *combinatorial*.
