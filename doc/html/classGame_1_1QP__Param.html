◆ dataCheck()

bool Game::QP_Param::dataCheck ( bool  forcesymm = true) const
private

Check that the data for the QP_Param class is valid Always works after calls to QP_Param::size() Checks that are done:

  • Number of columns in Q is same as Ny (Q should be square)
  • Number of columns of A should be Nx
  • Number of columns of B should be Ny
  • Number of rows in C should be Ny
  • Size of c should be Ny
  • A and B should have the same number of rows, equal to Ncons
  • if forcesymm is true, then Q should be symmetric
Returns
true if all above checks are cleared. false otherwise.
Parameters
forcesymmCheck if QP_Param::Q is symmetric

◆ size()

unsigned int Game::QP_Param::size ( )
private

Calculates Nx, Ny and Ncons Computes parameters in QP_Param:

  • Computes Ny as number of rows in QP_Param::Q
  • Computes Nx as number of columns in QP_Param::C
  • Computes Ncons as number of rows in QP_Param::b, i.e., the RHS of the constraints

For proper working, QP_Param::dataCheck() has to be run after this.

Returns
Ny, Number of variables in the quadratic program, QP

unique_ptr< GRBModel > Game::QP_Param::solveFixed ( arma::vec  x)

Given a value for the parameters in the definition of QP_Param, solve the parameterized quadratic program to optimality.

In terms of game theory, this can be viewed as the best response for a set of decisions by other players.
